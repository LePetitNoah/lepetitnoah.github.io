Minecraft Mod Dependencies

Terminé
2026 - Present

Description

Comme je développe actuellement un modpack pour Minecraft, je rencontre parfois des difficultés liées aux dépendances de certains mods. Il arrive notamment que des dépendances persistent même après la suppression d’un mod. Pour remédier à cela, j’ai entrepris de créer un petit outil en Python capable d’identifier et de visualiser les relations de dépendance entre les mods que j’utilise.

Pour ce faire, je m’appuie sur l’API proposée par CurseForge afin de récupérer la liste de mes mods ainsi que leurs dépendances associées. Les données pertinentes sont ensuite enregistrées dans des fichiers, organisés selon différents critères comme la version de Minecraft. À partir de ces informations, je génère une page web en HTML intégrant un graphe interactif grâce à PyVis Network. Il est également possible de consulter ces données sous forme d’arborescence directement dans la console, en utilisant la bibliothèque Rich. Plus récemment, j’ai ajouté un script Python qui lance un serveur local, permettant d’accéder à une interface utilisateur pour rechercher plus facilement les dépendances d’un mod.

Défis & Solutions

Défi

La principale difficulté à laquelle j’ai été confronté concerne l’API de CurseForge, qui ne permet pas d’obtenir les informations d’un mod à partir de son nom. Il faut en effet utiliser son “slug”, ce qui pose problème puisque celui-ci ne correspond pas toujours au nom du mod.

Solution trouvée

Pour contourner cette contrainte, j’ai développé un script supplémentaire destiné à retrouver le slug à partir du nom du mod, de la version de Minecraft et du type de mod loader utilisé. Grâce à ces paramètres, je peux réduire les résultats à une sélection restreinte de mods, affichée dans la console avec les informations essentielles. Il me suffit ensuite d’identifier le bon mod, de copier son slug et de récupérer ses dépendances.
